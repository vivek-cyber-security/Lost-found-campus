## ⚙️ HOW TO RUN IN XAMPP

### Step 1 — Install XAMPP
Download from: https://www.apachefriends.org/
Install and open XAMPP Control Panel.

### Step 2 — Start Services
- Click **Start** next to **Apache**
- Click **Start** next to **MySQL**

### Step 3 — Copy Project Files
Copy the entire `lost-found-campus` folder to:
```
C:\xampp\htdocs\lost-found-campus\
```

### Step 4 — Create the Database
1. Open your browser and go to: http://localhost/phpmyadmin
2. Click **New** (top left)
3. Name it: `lost_found_campus` → Click **Create**
4. Click the database name in the left panel
5. Click the **SQL** tab at the top
6. Open `database.sql` from the project folder
7. Paste the full SQL content → Click **Go**

> ✅ This creates all 3 tables and adds sample test data.

### Step 5 — Configure DB Credentials (if needed)
Open `db.php` and update:
```php
define('DB_HOST', 'localhost');
define('DB_USER', 'root');    // your MySQL username
define('DB_PASS', '');        // your MySQL password (usually blank in XAMPP)
define('DB_NAME', 'lost_found_campus');
```

### Step 6 — Run the Project
Open browser: http://localhost/lost-found-campus/

## ✨ FEATURES SUMMARY

| Feature | Implementation |
|---|---|
| User Registration & Login | PHP Sessions + bcrypt password_hash |
| Report Lost Item | PHP form → MySQL insert |
| Report Found Item | PHP form → MySQL insert |
| Smart Matching | Keyword comparison (PHP strpos + custom keywords) |
| AJAX Loading | fetch() in script.js → getMatches.php |
| XML Storage | PHP DOMDocument writes xml/matches.xml |
| Item Search Filter | JavaScript live filter on dashboard |
| Form Validation | JavaScript + PHP server-side |
| Responsive Design | CSS Flexbox + Grid |
| Clean UI | Poppins font, card layout, gradients |

---

## 🔧 TECHNOLOGIES USED
- **PHP** — Backend logic, sessions, DB queries
- **MySQL** — Data storage (users, lost_items, found_items)
- **HTML5** — Page structure
- **CSS3** — Styling (single style.css, no Bootstrap)
- **JavaScript** — Validation, AJAX fetch, filter
- **AJAX** — Dynamic match loading without page refresh
- **XML** — Match results stored and read from matches.xml
